# Representation Overview

OMNIABASE exists because a structure should not be confused with one representation of it.

Humans usually see numbers in base 10.

Base 10 is useful.

Base 10 is not ontologically privileged.

The same object may expose different surface properties under different representations.

OMNIABASE asks:

    what remains stable when the representation changes?

---

## Core path

    object -> representation -> base shift -> observation -> invariant residue

Meaning:

| Stage | Role |
|---|---|
| object | The thing being observed |
| representation | The current encoding or frame |
| base shift | The controlled change of representation |
| observation | Extracted feature, trace, property, or pattern |
| invariant residue | What remains stable across the shift |

---

## Why representation matters

If a property appears only in one representation, it may be representation-bound.

If a property survives many representations, it becomes structurally more interesting.

This does not automatically make it true.

It makes it more stable under the declared observational regime.

---

## Relation to OMNIA

OMNIABASE can provide representation-aware observations.

OMNIA can measure structural behavior.

OMNIA-VALIDATION can test artifacts and claims.

The separation is intentional:

    OMNIABASE observes representations.
    OMNIA measures structure.
    OMNIA-VALIDATION validates artifacts.
